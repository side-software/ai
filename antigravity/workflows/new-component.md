# New Component

Create a new component following the project's conventions.

## Steps

1. Ask what the component should be called and what it does.
2. Determine where it belongs based on the project's directory structure.
3. Read 2-3 existing components in the target directory to match patterns (imports, styling, state management, exports).
4. Create the component file following the observed conventions.
5. Add the component to relevant barrel exports if the project uses them.
6. Run the verify command to confirm 0 errors.
7. If the project uses a component catalog (Storybook, etc.), create a story or example.
